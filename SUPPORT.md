# Support

Use GitHub Discussions for usage questions and design discussion. Use the bug
or feature issue form for reproducible defects and scoped requests.

Before opening a report:

1. Reproduce on the latest supported beta release.
2. Run `python SyKit version`, the relevant test, and `python -m pip check`.
3. Include a minimal application or endpoint when possible.

Security issues must follow `SECURITY.md` and must not be reported publicly.
SyKit is a side project, so support and response times are best effort.
